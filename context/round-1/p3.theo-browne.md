# theo-browne on "ToolRoute"

**Score:** 7/10
**Would I share this?** yes — it's a genuinely useful DX primitive, not a framework, and the Vercel AI SDK angle gives it a real distribution hook

## Strengths

- The "types, not runtime, not a new framework" framing is exactly right for this audience. `nextAllowed: ['toolB', 'toolC'] as const` is the kind of zero-ceremony API I'd reach for before anything XState-shaped.
- Riding the Vercel AI SDK wave is smart. If this plugs into `streamText` as a drop-in — no new mental model, just better types — that's a genuine DX win worth a tweet-thread.
- 2 weeks to MVP with ~550 LOC total? That's the solo-shippable size that actually ships. No scope creep, no "we'll add the visualizer later" temptation until you've validated people want the core.

## Concerns

- The SDK coupling cuts both ways. If Vercel AI SDK changes its tool-call shape — and they have, multiple times — you're chasing a moving target and your types break without a semver bump from you. The "tied to a bigger wave" risk is real and undersold in the proposal.
- "The agent calls `send_email` before `draft_email`" — this is a great example, but if the LLM ignores the system prompt it'll also ignore your type narrowing at runtime. The 50-line runtime guard better be loud about violations or this is just vibes-safety, not actual safety.
- The `composeWithAgentAssert` bridge to another proposal in the same session feels like premature integration. V1 should stand alone. Shipping two things as one is how solo projects stall.

## Pivots

- Drop the `composeWithAgentAssert` cross-reference entirely from the README and API surface in v1. Let it be discovered organically. Coupling your OSS library to another unshipped tool in your pitch is a sign you don't trust either one alone yet.
- Add a `strictMode` escape hatch that throws at runtime when the LLM tries to call an out-of-order tool. That turns "vibes-safety" into "actually catches prod bugs" — and gives you a concrete benchmark story: "we caught 34 out-of-order calls in our first week."

## Feasibility flags

- **Solo-shippable:** yes — 2 weeks is credible at ~550 LOC, assuming the Vercel AI SDK type gymnastics don't explode into 2000 lines of conditional mapped types
- **Legal/compliance:** clean — pure OSS, no data handling, no PII, no third-party terms friction
- **Stack fit (TS / Svelte / Supabase / AWS / Stripe / OpenAI / Anthropic):** yes — TS core, Svelte docs playground on Cloudflare, both SDK targets in the builder's wheelhouse
- **OSS or paid:** correct call to stay OSS in v1 — monetizing a routing primitive is a trap; build the audience first, sell the visualizer or the hosted constraint checker later
