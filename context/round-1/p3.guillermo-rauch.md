# guillermo-rauch on "ToolRoute"

**Score:** 7/10
**Would I share this?** maybe — the primitive instinct is right, but it's Vercel AI SDK-specific and risks feeling like a patch rather than a first-class citizen

## Strengths

- Nails the actual footgun. Tool ordering isn't an XState problem, it's an HTTP-routing problem at the AI layer. `nextAllowed` as a const tuple is the right shape — composable, greppable, zero-framework.
- Fits the SDK's grain. The output is still the standard Vercel AI SDK `tools` object handed to `streamText`. Zero API-surface conflict. This is how good ecosystem extensions should work: invisible seam.
- Two-week OSS callling card is exactly right. The docs playground on Cloudflare Pages with a live example closes the loop in a way a README never could. That's shipping.

## Concerns

- "Plugin" is too strong a word for what this is. It's a wrapper + type-narrowing utility. If Vercel AI SDK ships a first-party `allowedTools` option in the next minor — which is plausible — this gets absorbed or orphaned overnight.
- The runtime guard is 50 lines of throw-on-violation. That's fine in dev, but agents in prod need graceful fallback, not a crash. What does the SDK do when the guard rejects the next tool call mid-stream? That's an open UX question with real production implications.
- `nextAllowed` is static — it can't express conditional branching. After `search`, maybe `read` is only allowed if the result was non-empty. Real pipelines hit this within days. The v1 constraint is fine, but if this isn't the stated v2 roadmap item, the tool will feel incomplete fast.

## Pivots

- Rename the core export from `defineTool` to `routedTool` and lean into the router metaphor explicitly. Then `createRouter(tools)` becomes `AgentRouter` — a router for AI tools. This makes the docs, README hero, and playground narrative coherent. "What if AI tools had routes?" is a shareable sentence. "What if tools had a nextAllowed array?" is not.
- Ship the `composeWithAgentAssert` interop on day one, not as a bonus. The joint story — ToolRoute handles the legality graph, AgentAssert handles invariants — is stronger than either standalone. Even a two-sentence callout in the README pointing to a sibling package elevates both. Ecosystem > standalone utility.

## Feasibility flags

- **Solo-shippable:** yes — 2 weeks is credible; the type machinery is the hard part and ~500 LOC is realistic for someone with conditional/mapped type fluency
- **Legal/compliance:** clean — pure OSS, no data handling, no external calls, no scraping
- **Stack fit (TS / Svelte / Supabase / AWS / Stripe / OpenAI / Anthropic):** yes — TS core + Svelte playground is exactly the stack; OpenAI/Anthropic adapters are implicit via Vercel AI SDK compatibility
- **OSS or paid:** OSS is correct — the value is adoption and ecosystem credibility; a paid tier here would kill the distribution before it starts
