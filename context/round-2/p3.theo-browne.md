# theo-browne on "ToolRoute"

**Score:** 8/10
**Would I share this?** yes — `createRouterFromTools` is a concrete, tweetable primitive; the 50-line runtime guard gives it a spine that makes the type story credible

## Strengths

- The single-source-of-truth pivot (`createRouterFromTools` derives tool names from the const array) directly kills the rename-drift footgun that makes similar tools feel half-baked — this is the kind of DX detail that actually matters
- Runtime guard built and demoed first, types layered on top: this is exactly the right order of operations. Type theater is dead on arrival; operational proof before compile-time decoration is what separates a real tool from a vibes library
- Scope is ruthlessly narrow — 500 LOC, Vercel AI SDK only, no `composeWithAgentAssert`, no paid tier, two weeks. This is a project that could actually ship and actually get used before the author moves on

## Concerns

- Vercel AI SDK coupling is load-bearing and the SDK's internal tool-call shape has changed at least twice in the last 12 months — a minor upstream API change could snap the narrowing entirely and there is no abstraction layer to absorb it; the README acknowledging it is fine but the maintenance surface is real
- The playground on Cloudflare Pages is a nice touch but a live playground for a routing library that wraps a streaming SDK is genuinely hard to wire up correctly in a browser — if it is half-broken it actively hurts first impressions; a static recorded terminal session in the README is probably more durable than a live demo
- `nextAllowed` as a per-tool const tuple means the routing graph lives scattered across individual tool definitions rather than in one readable place — for a three-tool agent this is fine, but at seven or eight tools the mental model inverts and you have to grep every tool to understand the graph; a `createRouterFromTools` visualizer mode (even just a console.log graph dump) would patch this

## Pivots

- Ship a `printGraph(router)` debug export that console.logs the routing graph as a human-readable adjacency list — one evening of work, zero deps, and it directly addresses the "scattered nextAllowed" readability gap without adding a full visualizer
- Before the Cloudflare playground, publish the recorded terminal session as a tweet thread: runtime guard rejecting `commit` before `review`, red squiggle in the editor, the whole flow — tweet gets you distribution signal before you spend time on playground infra

## Feasibility flags

- **Solo-shippable:** yes — 2 weeks is credible for 500 LOC + runtime guard + README + static docs page
- **Legal/compliance:** clean — pure OSS, no data collection, no SaaS surface in v1
- **Stack fit (TS / Svelte / Supabase / AWS / Stripe / OpenAI / Anthropic):** yes — TS core, Svelte docs page on Cloudflare Pages, Vercel AI SDK adapter; all in-stack
- **OSS or paid:** correct call — OSS calling card with a tight scope; paid tier would be a distraction at this stage and the audience (solo devs, small teams) needs to trust it first
